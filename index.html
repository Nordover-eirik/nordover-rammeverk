Sett opp i Elementor

Gå til Elementor → Settings → Editor V4 → aktiver. Åpne editoren → Class Manager → Variables. Opprett alle variabler i tabellene over. Bruk nøyaktig de samme navnene — da fungerer klassene i neste steg uten endring.

Alternerende seksjoner

For visuell rytme, veksle mellom bg-white og bg-light på annenhver seksjon. Kombinert med tekst-bilde left/right gir dette en naturlig flyt nedover siden uten at det trenger ekstra design.

Navngivingskonvensjon Konsistens på tvers

Alle klassenavn i dette rammeverket følger samme format. Bruk det på alle kundeprosjekter.

/* Mønster: [kategori]-[variant]-[modifier] */ /* Typografi */ heading-xl, heading-l, body-default, label /* Layout */ flex-row, flex-col, flex-center, flex-between /* Spacing */ gap-s, gap-m, section-m, section-l /* Container */ container-boxed, container-narrow, container-wide /* Bakgrunn */ bg-white, bg-light, bg-dark, bg-primary /* Knapper */ btn-primary, btn-secondary, btn-ghost, btn-small /* Kort */ card, card-hover, card-flat /* Bilder */ img-cover, img-rounded, img-circle /* Grid */ grid-2, grid-3, grid-4, grid-1-1, grid-2-1 /* Effekter */ shadow-soft, shadow-medium, divider

Regler

  • Alltid lowercase med bindestrek
  • Beskrivende funksjon — aldri utseende (card ikke gray-box)
  • Størrelser: xs / s / m / l / xl / 2xl / 3xl
  • Aldri kundenavn i klassenavn — rammeverket er universelt
  • Kundespesifikke tilpasninger: lag ny klasse med prosjekt-prefiks kun hvis nødvendig (f.eks. fjord-hero), men prøv å løse det med variabelendringer først
⚠️ Aldri gjør dette

blåBoks — norsk + camelCase + utseende-basert. section1 — sier ingenting. mt-32px — hardkodet pixel, bruk spacing-variabel. fjordShippingButton — kundespesifikt + camelCase. Alle disse er btn-primary, section-m, card i stedet.

V4-regelen

Sett aldri responsive verdier lokalt på et element. Alt går i Global Class. Da arver alle elementer med klassen de responsive reglene automatisk. Trenger du et unntak, bruk lokal class (rosa) — men vurder om du egentlig trenger en ny global class i stedet.

Arbeidsflyt Brief → lansering

Samme rekkefølge for hvert prosjekt. De første 2–3 timene på fundament sparer dager i bygging.

1
Variabler Kopier variabel-listen fra dette rammeverket inn i Elementor. Bytt primary, font-heading, font-body med kundens verdier. Resten beholder du som wireframe-defaults.
2
Klasser Opprett alle Global Classes fra steg 2. De refererer til variablene du nettopp satte opp. Inkluder responsive verdier per breakpoint i hver klasse.
3
Komponenter Bygg header, footer, hero, CTA, tjenestekort og grid-seksjoner som V4 Components. Sett opp Component Properties for redigerbare felt (tekst, bilder, lenker).
4
Wireframe-bygging Bygg alle sider med nøytral palett. Fokus på struktur, spacing, informasjonsarkitektur. Presenter for kunde som wireframe.
5
Merkeidentitet Oppdater Variables med kundens farger, fonter, og border-radius. Juster knapp-stiler, bakgrunnsfarger. Hele nettsiden transformeres automatisk.
6
Kvalitetskontroll Class Manager-audit, responsive test, PageSpeed, lenkesjekk, skjematest.
7
Overlevering Kunden redigerer kun innhold via Component Properties. Designsystemet er låst og konsistent.

Sjekkliste Før lansering

Klikk for å huke av. Lagres ikke — bruk som visuell gjennomgang per prosjekt.

Designsystem

  • Alle fargevariabler er definert — ingen hardkodede hex i klasser
  • Typografi-variabler satt opp med kundens fonter + fallbacks
  • Spacing bruker variabler — ingen manuelle pixel-verdier
  • Alle knappvarianter har hover-, focus- og active-states
  • Class Manager — ingen ubrukte eller duplikate klasser
  • Navngivingskonvensjon fulgt konsistent

Responsivt

  • Alle sider testet på desktop, tablet og mobil
  • Font-størrelser justert per breakpoint i klassene
  • Seksjons-padding redusert for tablet og mobil
  • Grid-kolonner kollapser riktig (4→2→1, 3→2→1)
  • Bilder har riktig aspect-ratio på alle enheter
  • Ingen skjulte duplikatelementer — alt løst med responsive stilregler
  • Touch-targets min 44×44px på knapper og lenker (mobil)

Ytelse

  • PageSpeed mobil score 85+
  • Bilder optimalisert — WebP, lazy loading, riktige dimensjoner
  • V4-elementer brukt der de finnes — V3 kun som fallback
  • Ingen inline CSS — alt i klasser/variabler
  • Fonter preloaded eller system stack brukt

Innhold & SEO

  • Alle lenker fungerer
  • Skjema tester OK — suksess/feil-states vises riktig
  • SEO-titler og meta utfylt på alle sider
  • Favicon og OG-bilder satt opp
  • Component Properties eksponert for kunden
  • Ikke-admin brukere kan ikke endre Global Classes/Variables
